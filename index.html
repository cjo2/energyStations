    This web application pulls data from two sources: National Renewable Energy Laboratory and mygasfeed.com. You can click on each marker to get more information about each station. You may also click on table rows to center the map on that location as well as open an info window. The map and table elements are hidden when first starting the web application and the focus is instantly set on the search bar. This is for two reasons: 1) the user knows that he or she must type in a location to continue 2) the user may begin typing as soon as the website loads. The tables only become fully visible when the data is received and processed. I have not bothered to check if there are applications that are similar, though I am confident they exist (I know they do as apps on iOS). I just wanted to be able to build this without taking influence from other sources.
    * mid and prem prices are currently not functional. with the current api, multiple ajax calls must be done (after getting the initial station info).
